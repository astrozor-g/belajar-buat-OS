Sebelum nulis kode, lo harus ngerti cara kerja mesin

Apa itu OS?

OS adalah jembatan antara hardware dan software. Tanpa OS, program gak bisa ngomong ke hardware secara langsung dengan aman.

Lapisan Sistem
  ┌─────────────────────┐
  │   Aplikasi (Chrome) │  ← lo pakai ini sehari-hari
  ├─────────────────────┤
  │   OS (Linux/Win)    │  ← yang mau kita bikin!
  ├─────────────────────┤
  │   Hardware (CPU)    │  ← fisik
  └─────────────────────┘

Sistem Bilangan Biner

CPU hanya mengerti 0 dan 1 (listrik mati/nyala). Semua data — teks, gambar, video — pada akhirnya adalah deretan 0 dan 1.

Konversi Desimal → Biner

Contoh: angka 13 dalam biner:

Biner
  13 ÷ 2 = 6  sisa 1  ← LSB (bit paling kanan)
   6 ÷ 2 = 3  sisa 0
   3 ÷ 2 = 1  sisa 1
   1 ÷ 2 = 0  sisa 1  ← MSB (bit paling kiri)

  Baca dari bawah ke atas: 1101
  Jadi 13 (desimal) = 1101 (biner)

  Verifikasi: 8+4+0+1 = 13 ✓

Hexadecimal (Hex)

Hex pakai 16 simbol (0-9, A-F). Dipakai programmer OS karena lebih ringkas dari biner.

DesimalBinerHex
000000x0
1010100xA
1511110xF
255111111110xFF
💡
Kenapa hex penting di OS? Alamat memori selalu ditulis dalam hex. Contoh: 0xFFFF0000 adalah alamat di mana CPU mulai boot. Lo akan sering lihat ini!

Logika Boolean

Tiga operasi dasar yang jadi fondasi semua logika di komputer:

OperasiSimbolArtiContoh
AND&&Dua-duanya harus benar1 AND 1 = 1
OR||Salah satu boleh benar0 OR 1 = 1
NOT!KebalikanNOT 1 = 0

Angka 10 dalam desimal, berapa dalam biner?
🔴 1001
🟢 1010

✅ Benar! 10 = 8+2 = 1010 dalam biner. 8=1000, 2=0010, jumlahnya 1010!

Level 1 — Linux & Terminal
OS yang akan lo bikin nanti berjalan di atas Linux tools

Kenapa Linux?

Semua SBC (Raspberry Pi, OrangePi, dll) pakai Linux sebagai OS default. Tools untuk build OS (compiler, linker, debugger) semua ada di Linux. Makanya lo wajib familiar sama Linux.

✅
Good news: Lo gak perlu install Linux di laptop/PC! Bisa pakai WSL2 di Windows, atau Virtual Machine, atau langsung di SBC lo nanti.

Perintah Terminal Wajib

BASH
# Navigasi
pwd           # lihat posisi sekarang
ls -la         # lihat semua file
cd /home/user  # pindah folder
cd ..          # naik satu level

# File operations
mkdir osproject    # buat folder
touch main.c       # buat file kosong
cp file1 file2     # copy file
mv file1 folder/   # pindah/rename
rm file.txt        # hapus file

# Lihat isi file
cat main.c         # tampilkan isi file
nano main.c        # edit file (simple)

# Info sistem
uname -a           # info kernel Linux
lscpu              # info CPU
free -h            # info RAM

Struktur Folder Linux

Filesystem
  /                 ← root (puncak semua folder)
  ├── bin/          ← program dasar (ls, cat, dll)
  ├── boot/         ← bootloader & kernel ← PENTING buat OS dev!
  ├── dev/          ← device files (hardware)
  ├── etc/          ← konfigurasi sistem
  ├── home/         ← folder user (lo kerja di sini)
  ├── lib/          ← library sistem
  ├── proc/         ← info proses yang lagi jalan
  └── sys/          ← interface ke kernel

Bash Scripting Dasar

Bash script itu file teks berisi perintah Linux yang bisa dijalankan otomatis. Nanti lo bakal pakai ini buat build system OS lo.

BASH
#!/bin/bash
# Baris pertama wajib ada (shebang)

# Variable
NAMA="OSku"
VERSI="0.1"

# Output
echo "Building $NAMA v$VERSI..."

# Kondisi
if [ -f "kernel.c" ]; then
    echo "File kernel.c ditemukan!"
else
    echo "File tidak ada!"
fi

# Loop
for file in *.c; do
    echo "Compiling: $file"
    gcc -c "$file"
done

echo "Build selesai!"

Proses di Linux

Setiap program yang jalan di Linux itu disebut proses. OS lo nanti harus bisa manage proses-proses ini.

BASH
ps aux         # lihat semua proses
top            # monitor proses real-time
kill 1234      # hentikan proses dengan PID 1234
jobs           # lihat proses background
⚠️
Catat ini! Setiap proses punya PID (Process ID) unik. OS lo nanti harus assign PID ke setiap proses yang dijalankan — ini namanya process management.

Pointer — Konsep Terpenting

Pointer adalah variabel yang menyimpan alamat memori. Ini konsep paling penting di C dan OS development karena OS harus langsung manipulasi memori.

C
#include <stdio.h>

int main() {
    int nilai = 42;      // variabel biasa
    int *ptr = &nilai;   // pointer ke 'nilai'
                         // & = ambil alamat memori

    printf("Nilai: %d\n", nilai);    // 42
    printf("Alamat: %p\n", ptr);    // misal: 0x7ffd1234
    printf("Via pointer: %d\n", *ptr); // 42 juga
                         // * = baca nilai di alamat itu

    *ptr = 100;   // ubah nilai lewat pointer
    printf("Sekarang: %d\n", nilai);  // 100!

    return 0;
}

Pointer di OS — Contoh Nyata

Di OS development, lo pakai pointer buat langsung nulis ke alamat hardware:

C
// Contoh: nulis ke UART (serial port) di SBC
// Alamat UART di Raspberry Pi = 0x3F201000

volatile uint32_t *uart = (volatile uint32_t*)0x3F201000;

// Kirim karakter 'A' lewat UART
*uart = 'A';

// Ini cara OS "ngobrol" langsung sama hardware!
⚠️
volatile dipakai supaya compiler gak "optimize away" akses ke hardware register. Ini wajib di OS dev!

Memory Management

C
#include <stdlib.h>

int main() {
    // malloc = minta memori dari heap
    int *arr = (int*) malloc(5 * sizeof(int));

    if (arr == NULL) {
        printf("Memori habis!\n");
        return 1;
    }

    // Isi array
    for (int i = 0; i < 5; i++) {
        arr[i] = i * 10;
    }

    // WAJIB free setelah selesai!
    free(arr);

    return 0;
}
🚨
Memory Leak! Kalau lo malloc tapi lupa free, memori "bocor" dan lama-lama habis. Di OS dev, ini sangat kritis — OS lo bisa crash!

Struct — Bikin Tipe Data Sendiri

C
// Contoh: struct buat proses (kayak di OS beneran!)
typedef struct {
    int   pid;           // Process ID
    char  name[32];     // Nama proses
    int   state;         // 0=ready, 1=running, 2=blocked
    void *stack_ptr;     // Pointer ke stack proses
} Process;

int main() {
    Process p1;
    p1.pid = 1;
    p1.state = 1;  // running

    printf("PID: %d\n", p1.pid);
    return 0;
}

Level 3 — Computer Architecture
Memahami hardware yang akan lo kendalikan

ARM Architecture

Semua SBC (Raspberry Pi, OrangePi, RockPi) pakai CPU ARM. ARM berbeda dari Intel/AMD (x86) yang ada di laptop/PC biasa.

FiturARM (SBC)x86 (PC/Laptop)
DayaHemat (1-5W)Boros (65-200W)
InstruksiRISC (simpel)CISC (kompleks)
Harga chipMurahMahal
Buat OS dev pemula✅ Lebih mudahLebih rumit

Register CPU

Register itu memori super kecil yang ada di dalam CPU sendiri — paling cepat diakses. ARM 64-bit punya register:

ARM64
  x0 - x7    → Parameter fungsi & return value
  x8 - x18   → General purpose (pakai bebas)
  x19 - x28  → Callee-saved (harus dijaga)
  x29        → Frame Pointer (FP)
  x30        → Link Register (LR) — alamat balik fungsi
  sp         → Stack Pointer — posisi stack sekarang
  pc         → Program Counter — instruksi berikutnya

Memory Map di SBC

Di SBC, setiap hardware punya "alamat" di memori. OS lo nanti nulis ke alamat itu buat kontrol hardware:

Raspberry Pi 4 Memory Map
  0x00000000 - 0x3FFFFFFF  → RAM (1GB)
  0xFC000000 - 0xFCFFFFFF  → Peripheral (hardware)
    ↳ 0xFC201000           → UART0 (serial)
    ↳ 0xFC200000           → GPIO
    ↳ 0xFC00B000           → Interrupt Controller
  0xFF800000+              → VideoCore (GPU)

Interrupt — Cara Hardware "Manggil" OS

Interrupt adalah sinyal dari hardware ke CPU: "Hei, ada sesuatu yang butuh perhatian!"

Contoh Interrupt
  Keyboard ditekan  →  Keyboard Controller kirim IRQ
                    →  CPU stop program sekarang
                    →  CPU jalankan "Interrupt Handler"
                    →  Handler proses keystroke
                    →  CPU lanjut program tadi

  Tanpa interrupt: OS harus terus-terusan nanya ke
  hardware "ada input gak? ada input gak?" — BOROS!
💡
Interrupt Handler adalah fungsi yang lo tulis di OS lo buat handle event dari hardware. Ini salah satu hal pertama yang lo implement waktu bikin OS!

Level 4 — Assembly ARM
Ngobrol langsung sama CPU dalam bahasa aslinya

Apa itu Assembly?

Assembly adalah bahasa pemrograman yang instruksinya 1:1 dengan instruksi CPU. Setiap baris Assembly = 1 operasi CPU.

LevelContohKecepatan
Pythonx = 5 + 3Lambat
Cint x = 5 + 3;Cepat
AssemblyMOV x0, #5 / ADD x0, x0, #3Tercepat

Instruksi ARM Dasar

ARM64 Assembly
// MOV — pindahkan nilai ke register
MOV  x0, #42       // x0 = 42
MOV  x1, x0        // x1 = x0

// Aritmatika
ADD  x0, x1, x2    // x0 = x1 + x2
SUB  x0, x1, #5   // x0 = x1 - 5
MUL  x0, x1, x2    // x0 = x1 * x2

// Load & Store (akses memori)
LDR  x0, [x1]      // x0 = *x1 (baca dari memori)
STR  x0, [x1]      // *x1 = x0 (tulis ke memori)

// Branch (lompat)
B    label          // lompat ke label
BL   fungsi         // panggil fungsi (simpan return addr)
RET                 // kembali dari fungsi

// Kondisi
CMP  x0, #10       // bandingkan x0 dengan 10
BEQ  label          // lompat jika sama
BNE  label          // lompat jika tidak sama

Contoh: Bootloader Sederhana

Ini adalah kode pertama yang dijalankan saat SBC lo nyala:

ARM64 — boot.S
// File: boot.S
// Kode ini dijalankan PERTAMA KALI saat SBC nyala

.section ".text.boot"  // section khusus boot
.global _start           // entry point

_start:
    // CPU mulai di core 0, matikan core lain
    MRS  x1, mpidr_el1     // baca CPU ID
    AND  x1, x1, #3        // ambil bit 0-1
    CBZ  x1, init          // kalau core 0, lanjut

halt:
    WFE                    // core lain: tidur
    B    halt

init:
    // Set Stack Pointer
    LDR  x1, =0x80000     // alamat stack
    MOV  sp, x1

    // Lompat ke kernel C!
    BL   kernel_main       // panggil fungsi C

    // Kalau kernel_main return (harusnya gak)
    B    halt
🎉
Keren kan? Setelah boot.S setup stack, dia panggil kernel_main() yang lo tulis dalam C. Itulah kenapa C + Assembly adalah combo sempurna — Assembly buat setup awal, C buat logika utama!

1. Process Management

OS harus bisa jalankan banyak program "sekaligus" padahal CPU cuma satu. Tricknya: bergantian super cepat!

C — Contoh PCB (Process Control Block)
// PCB = struktur data yang nyimpen info tiap proses
typedef struct {
    uint32_t pid;          // Process ID unik
    uint32_t state;        // READY/RUNNING/BLOCKED
    uint64_t registers[31];// State CPU saat di-pause
    uint64_t sp;           // Stack pointer
    uint64_t pc;           // Program counter
    uint64_t *page_table;  // Memori milik proses ini
} PCB;

// OS simpan semua proses dalam array/list
PCB proses_table[256];  // max 256 proses

2. Memory Management

OS harus ngatur siapa boleh pakai memori mana. Ini pakai sistem paging:

Konsep Paging
  Memori fisik dibagi jadi "halaman" (page) 4KB:

  Proses A pikir punya:  0x0000 - 0xFFFF (64KB)
  Proses B pikir punya:  0x0000 - 0xFFFF (64KB)

  OS translate via Page Table:
  A's 0x0000 → fisik 0x10000
  B's 0x0000 → fisik 0x30000

  Jadi tiap proses "terisolasi" — A gak bisa
  akses memori B. Inilah KEAMANAN OS!

3. Simple Kernel Main

Inilah "isi" kernel pertama lo — fungsi C yang dipanggil dari Assembly:

C — kernel.c
// Alamat UART Raspberry Pi 3
#define UART0_BASE  0x3F201000
#define UART0_DR    (*((volatile uint32_t*)(UART0_BASE + 0x00)))
#define UART0_FR    (*((volatile uint32_t*)(UART0_BASE + 0x18)))

void uart_putc(char c) {
    // Tunggu UART siap
    while (UART0_FR & (1 << 5));
    UART0_DR = c;
}

void uart_puts(const char *str) {
    while (*str) uart_putc(*str++);
}

// Fungsi ini dipanggil dari boot.S
void kernel_main() {
    uart_puts("Halo dari OS buatan gw sendiri!\n");
    uart_puts("Kernel berhasil booting!\n");

    // Loop selamanya (OS gak boleh berhenti)
    while(1);
}
🎉
Ini OS pertama lo! boot.S + kernel.c ini sudah cukup buat bikin OS minimal yang bisa print teks ke serial port di Raspberry Pi. Dari sini lo tinggal tambah fitur satu per satu!
